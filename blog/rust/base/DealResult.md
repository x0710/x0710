# 关于`Result`及`Option`方法总结
- `unwrap()`            如果结果错误，直接引发恐慌
- `unwrap_or()`         如果结果错误，返回默认值
- `unwrap_or_else()`    如果结果错误，再通过闭包进行处理并返回值
- `map()`               对结果进行映射，如果结果错误，则不做处理，依旧返回错误
- `map_or()`            当结果错误时,返回默认映射
- `map_or_else()`       对每个结果进行闭包映射，包括结果错误时

- `as_ref()`    对结果的值进行取引用，若为`None`则结果依旧为`None`，但地址不同。
- `as_deref()`    ~~在`as_ref()`的基础上再把包裹值转变为需要的类型~~
