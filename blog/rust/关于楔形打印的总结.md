# 
## 原题
```
本题要求你写个程序把给定的符号打印成沙漏的形状。例如给定17个“*”，要求按下列格式打印

*****
 ***
  *
 ***
*****

所谓“沙漏形状”，是指每行输出奇数个符号；各行符号中心对齐；相邻两行符号数差2；符号数先从大到小顺序递减到1，再从小到大顺序递增；首尾符号数相等。

给定任意N个符号，不一定能正好组成一个沙漏。要求打印出的沙漏能用掉尽可能多的符号。
输入格式:

输入在一行给出1个正整数N（≤1000）和一个符号，中间以空格分隔。
输出格式:

首先打印出由给定符号组成的最大的沙漏形状，最后在一行中输出剩下没用掉的符号数。
```

## 感悟
### 字串处理
1. 标准输入流
- rust 的标准输入流通过`std::io` trait 提供，采用`io::stdin()`以获取标准输入
- 通过使用`read_line()`方法从stdin读取整行字符串。
- **在切割字符串前，必须调用`trim()`删除两端换行符，否则会引发_恐慌_。**

2. 字符串切割
- 可以通过`split_whitespace()`方法对`String`类型数据，通过空白符进行切割。
该方法返回了一个迭代器，指向被分割的每部分。
- 可以通过`collect()`方法对迭代器中每个元素进行集合化，该函数需要指定目标集合种类。
- 可以通过`parse::<u32>()`方法对`&str`字符串切片进行转整数。

3. 一个字符串一次性转多个数字
方法`split_whitespace()`切割后，需要使用到_闭包_语句，还没有学习到，简单带过：
```rust
let mut input = String::new();
std::io.stdin().read_line(&mut input).unwrap();
let nums = input.trim()
            .split_whitespace()
            .map(|x| x.parse().unwrap()) // 这里用到了闭包
            .collect<Vec<u32>>();
```

### 循环
1. `Range` 的左值必须要小于等于右值
如果你想在循环时从大到小遍历，需要对`Range`调用`rev()`方法。
2. `Range` 的默认步长为1
如果你想换为其它步长，请调用`step_by()`方法！

### 语法糖
1. 
如果一个函数返回的值为`Option<&T>`且T没有实现`Copy Trait`
在接收返回值时，编译器会为我们自动加上解引用，无需手动添加

2. 
在`println!()`宏中，可以使用`{val}`(val为变量名)来直接打印变量！
但是请不要奢望像~`{val.n}`~这样的奇怪代码了好吗？
